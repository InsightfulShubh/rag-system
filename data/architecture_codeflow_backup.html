System Overview

Entry app/main.py
FastAPI application factory. Registers three routers (/api/ingest, /api/query, /api/sessions). Calls init_db() at startup to create SQLite tables.
Ingest Two-step pipeline
Read raw .txt → chunk (sliding window, 500 chars / 50 overlap) → batch embed (1 API call per file) → mean embedding for file-level index → save to JSON files.
Retrieval Two-stage RAG
Stage 1: NumPy vectorized cosine similarity against all file embeddings → top 2 files.
Stage 2: Lazy-load chunks for those 2 files only → top 5 chunks. ~40 comparisons total instead of 40k+.
Chat Agent Tool calling loop
LLM receives search_kb tool definition + conversation history. Forced to call the tool (FORCE_TOOL_USAGE=true). Backend executes retrieval, feeds result back. Second LLM call generates final answer.
Persistence SQLite + JSON
Chat sessions & messages: SQLite (data/chat.db). Embeddings: JSON files (data/embeddings/). Absolute paths used everywhere to avoid working-directory issues.
Providers Dual LLM support
Single USE_AZURE_OPENAI flag in .env switches between OpenAI public API and Azure OpenAI (EPAM DIAL). No code changes needed. Factory in clients/llm_client.py.

High-Level Component Map

Ingestion Pipeline Flow

Triggered by POST /api/ingest (single file) or POST /api/ingest-all (directory). Run once before querying.

1
HTTP Request received by route
routes/ingest.py → ingest_directory(request)
IN IngestDirectoryRequest { dir_path: str } (defaults to absolute data/raw/)
↓
2
DocumentReader.read_file(file_path)
storage/document_reader.py — reads raw .txt, UTF-8 with errors='replace'
IN file_path: str   OUT text: str (full file content)
↓
3
split_into_chunks(text)
utils/chunking.py — sliding window. Step = chunk_size − overlap = 500 − 50 = 450 chars per advance.
IN text: str   OUT chunks_text: list[str] (~N chunks of ≤500 chars each)
↓
4
get_embeddings_batch(chunks_text)
utils/embedding.py — ONE API call to OpenAI/Azure for all chunks of the file. Returns N vectors of 1536 floats each.
IN texts: list[str]   OUT embeddings: list[list[float]] shape (N, 1536)
↓
5
_mean_embedding(embeddings)
services/ingestion.py — np.mean(matrix, axis=0). Produces a single 1536-float vector representing the whole file.
IN list[list[float]] shape (N, 1536)   OUT file_embedding: list[float] (1536,)
↓
6
VectorStore.save_chunks(file_name, chunks)
storage/vector_store.py — writes data/embeddings/chunks/<file_name>.json. One JSON file per document.
IN file_name: str, chunks: list[{text, embedding}]   OUT JSON file on disk
↓
7
VectorStore.save_file_embedding(file_name, embedding)
storage/vector_store.py — appends/updates entry in data/embeddings/files.json. This is what Stage 1 reads.
IN file_name: str, embedding: list[float]   OUT files.json updated on disk
↓
8
Response returned
OUT IngestDirectoryResponse { files_processed, total_chunks, results, errors }

Single-Turn Query Flow — POST /api/query

Stateless. No session. No history. Embeds query → 2-stage retrieval → LLM answer.

1
HTTP request → route
routes/query.py → answer_query(request)
IN QueryRequest { query: str }
↓
2
RetrievalService.retrieve_and_answer(query)
services/retrieval.py
IN query: str
↓
3
get_embedding(query)
utils/embedding.py — single API call to OpenAI/Azure Embeddings.
IN text: str   OUT query_embedding: list[float] (1536 floats)
↓
4
Stage 1: _get_top_files(query_embedding, top_k=2)
Loads files.json → NumPy matrix (N_files × 1536) → vectorized cosine: scores = (matrix @ query_vec) / (norms_matrix × norm_query) → np.argpartition O(n) top-2 selection.
IN query_embedding: list[float], top_k=2   OUT top_files: list[str] e.g. ["Machine_Learning.txt", "Deep_Learning.txt"]
↓
5
Stage 2: _get_top_chunks(query_embedding, top_files, top_k=5)
Lazy-loads chunks/Machine_Learning.txt.json + chunks/Deep_Learning.txt.json. Computes cosine similarity across ~40 chunks. Returns top-5 merged across both files.
IN query_embedding, file_names: list[str], top_k=5   OUT top_chunks: list[{text, embedding, file_name}]
↓
6
LLMService.generate_answer(query, top_chunks)
services/llm.py — builds system + user messages with context injected. Calls chat.completions.create(). Temperature=0 (deterministic).
IN query: str, context_chunks: list[dict]   OUT answer: str
↓
7
Response returned
OUT QueryResponse { answer: str, sources: list[str] }

Chat Agent Flow — POST /api/sessions/{id}/messages

Stateful. OpenAI tool/function calling loop. History from SQLite. Forced KB search (FORCE_TOOL_USAGE=true by default).

1
HTTP request → route → ChatService.chat()
routes/session.py → send_message(session_id, request) → checks db.session_exists() → calls chat_service.chat(session_id, message)
IN session_id: str, MessageRequest { message: str }
↓
2
db.save_message(session_id, "user", message)
storage/db.py — INSERT into messages table with UUID id and UTC timestamp.
IN session_id, role="user", content: str   OUT {id, session_id, role, content, created_at}
↓
3
db.get_messages(session_id, limit=HISTORY_LIMIT)
storage/db.py — SELECT last 6 rows DESC then reversed to chronological order. This is the conversation context window.
IN session_id: str, limit=6   OUT history: list[{role, content, ...}] oldest first
↓
4
Build messages list for LLM
messages = [{role:"system", content: settings.system_prompt}, ...history]. The user message is already the last history item — not appended again.
OUT messages: list[{role, content}] shape: [system, user_1, assistant_1, …, user_N]
↓
5
LLM Call #1 — with tool definition
client.chat.completions.create(model, messages, tools=[SEARCH_KB_TOOL], tool_choice="required")
IN messages, tools=[SEARCH_KB_TOOL]
OUT (tool call) assistant_msg.tool_calls[0] = {id, function.name="search_kb", function.arguments='{"query":"..."}'}
↓
6
execute_tool_call(tool_call) → search_kb(query)
services/tools.py — parses tool_call.function.arguments (JSON string) → calls retrieval_service.search(query) → full 2-stage retrieval (no LLM).
IN tool_call object
OUT json.dumps({context: "[Source:…]\n…", sources: ["Machine_Learning.txt"]})
↓
7
Append tool result to messages
messages thread becomes: [system … user assistant(tool_call) tool(result)]. tool_call_id must match the request id — OpenAI enforces this pairing.
IN tool_call_id, content: json_string   OUT messages list extended
↓
8
LLM Call #2 — final answer
tool_choice="none" — LLM reads context from tool result and writes grounded text answer. Cannot call tool again.
IN messages (now includes tool result)   OUT assistant_msg.content: str (final answer text)
↓
9
db.save_message(session_id, "assistant", answer)
Persists the reply so it becomes part of history for the next turn.
OUT MessageResponse { answer: str, sources: list[str] }
