🗄️ SQLite Schema — data/chat.db (NEW in v2)

sessions
idPK TEXT UUID v4 string
created_at TEXT ISO datetime UTC
→
messages
idPK TEXT UUID v4 string
session_idFK TEXT → sessions.id ON DELETE CASCADE
role TEXT "user" or "assistant"
content TEXT message text
created_at TEXT ISO datetime UTC
• init_db() called at FastAPI startup — no-op if tables exist.
• get_messages(limit=6) — SELECT DESC then reverse → chronological order for LLM.
• session_exists(id) — used by routes to return 404 before calling services.
• CASCADE ensures deleting a session removes all its messages.
• row_factory = sqlite3.Row — allows row["column"] dict-style access.
