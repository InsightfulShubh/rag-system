Stage 1 — _get_top_files() NumPy · In-Memory
ItemDetail
Fileapp/services/retrieval.py
Method_get_top_files(query_embedding, top_k=2) → list[str]
Inputlist[float] — query vector (1536-dim)
Outputlist[str] — top 2 file names by similarity
Readsdata/embeddings/files.json — all file embeddings (preloaded in memory)
Libnumpy
Algorithm 1. Stack file embeddings into matrix (N×1536)
2. matrix @ query_vec — batch dot products
3. Divide by norms → cosine similarities
4. np.argpartition O(n) top-k selection

📁 Storage Layout

📄
data/embeddings/files.json
All file-level embeddings. Loaded into memory at startup. Shape: {filename: {embedding: [1536 floats]}}
📂
data/embeddings/chunks/<name>.json
Per-file chunk embeddings. Lazily loaded only when file is selected in Stage 1. Shape: [{text, embedding}, ...]
📝
data/raw/<*.txt>
Raw knowledge base text files. Input to ingestion pipeline.
